Research Paper

Abstract

This research investigates the convergence of traditional command injection vulnerabilities and emerging prompt injection techniques in distributed big data platforms. We analyze two critical vulnerabilities: CVE-2022-25168 in Apache Hadoop's FileUtil.unTar() API (CVSS 9.8) and SPARK-50239 in Apache Spark's executor configuration (CVSS 8.8). Through practical experimentation using Hadoop 3.3.6 and Spark 3.5.0, we demonstrate how attackers can chain command injection with prompt injection in RAG-based analytics interfaces. Our implementation includes attack scenarios for spark-submit JavaOptions injection, HDFS data poisoning, and P2SQL injection, alongside a defense framework combining static analysis, runtime monitoring, and semantic detection.
